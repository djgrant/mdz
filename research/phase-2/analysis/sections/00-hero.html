MDZ research · phase 2

Accountable execution

Phase 1 showed that models can execute MDZ programs. Phase 2 asks whether that execution can be held to account: real-world-shaped procedures, run agentically, with every trace captured by the harness rather than reported by the model.

E1 · strict mode 10/10 type halts

A pragma flips type violations from silent coercion to a clean halt. Syntax faults still execute 9/10 – the model repairs them in the act of reading.

 E2 · map-reduce skill 1.0 payload fidelity

Every delegation attempt carried its items and lambda verbatim. The failures were tool routing and worker refusal, both outside the notation.

 E3 · state store 0.25 of assigns journalled

Models executed correctly and ignored the journalling discipline, treating the store as documentation instead of memory. Negative result.

 E4 · procedure vs goal 18/20 vs 7/20 compliant

Where policy contradicts the helpful default, the MDZ procedure holds and the goal prompt folds. Both procedure failures were a checker artefact.
