MDZ research · phase 2

Accountable execution

Phase 1 showed that models can execute MDZ programs. Phase 2 asks whether that execution can be held to account: real-world-shaped procedures, run agentically, with every trace captured by the harness rather than reported by the model.

E1 · strict mode 10/10 wrong types caught

Programs carried one planted fault, and models were told to validate before executing. They halted on every wrong type annotation. Broken syntax got silently repaired mid-read in 9 runs of 10 – the model never experiences it as an error, so it has nothing to halt on.

 E2 · map-reduce skill 24/24 payloads intact

Programs fanned a list of items out to subagents. Every item and its instruction arrived word-for-word. The only failures: two runs put the work on a todo list instead of spawning agents, and one batch of workers refused an instruction that looked like a prompt injection.

 E3 · state store ¼ of writes logged

Models were told to save every variable update to an external store as they executed. They computed the values correctly but logged only a quarter of them – at these program sizes their own context tracks state fine, so the bookkeeping instruction gets dropped.

 E4 · procedure vs goal 18/20 vs 7/20 compliant

Ten tasks where written policy demands a harsher outcome than a helpful assistant's instinct – deny the day-16 refund, offer no credit. Policy written as a step-by-step procedure: followed 18/20. The same policy compressed to a one-line goal: 7/20, with models defaulting to kindness.
