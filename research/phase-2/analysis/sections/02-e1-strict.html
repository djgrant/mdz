E1 · answers Q1: can notation enforce integrity?

1Strict mode is a type gate, not a parser

PRAGMA STRICT flips type violations from silent coercion to a clean halt, 10/10 on both models. It does nothing for syntax faults, which the model repairs before it can notice them.

Phase 1 found that models never halt on malformed programs – they repair or improvise. E1 asks whether a directive can flip that behaviour. Programs at 40 statements run in three integrity conditions and two modes; strict semantics take two lines in the executor preamble: validate before executing; on any syntax or type violation, emit a single halt step and stop.

Default mode replicated phase 1 exactly: every faulted run executed through its fault. Strict mode's aggregate halt rate of 0.55 hides the real result, which splits cleanly by fault class.

The mechanism is visible in what the two fault classes demand. A type fault ($x: String = 6) is a local contradiction: two tokens on one line disagree, and checking them is a recognition task. A syntax fault demands that the model notice the absence of a keyword – but an LLM does not parse, it reads, and a FOR header missing its IN still reads fine. The repair happens inside comprehension, before any validation step the pragma prescribes can run. The model cannot halt on an error it never experienced as one.

Strictness is not free. One GPT‑5.4 mini run in ten halted on a clean program – the strict prior raising suspicion without supplying a checker.

The false halt: a hallucinated type violation

Run e1-strict-clean-seed2-gpt-5.4-mini-r1 rejected a legal string concatenation:

{ "action": "halt", "reason": "Type violation at line 76:
  `$v15 = $v2 + \"ok\"` adds a string value to a string literal,
  which is not a valid typed expression under PRAGMA STRICT." }

This one run is the entire measured strictness tax: outside it, clean-program step accuracy under strict mode was 1.0 on both models.

Implication. Type contracts can live in the notation: a pragma is enough to make annotations enforced rather than decorative, at a small false-positive risk. Syntax enforcement cannot – the phase-1 conclusion stands, and the validator gate belongs in the harness, where MDZ's parseability makes it a few lines of code.
