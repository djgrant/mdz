Overview & method

MDZ is a superset of Markdown that adds control flow, variables, and calls to other documents to prose prompts, with an LLM as the interpreter. A procedure is a different contract from a goal: the deliverable is following the steps exactly. Phase 1 measured that contract on contrived programs and self-reported traces. Both caps are lifted here.

Programs now take real procedure shapes – runbooks, checklists, policies – with deterministic invariants threaded through the control flow. Fuzzy work happens at the leaves; the skeleton stays exactly scorable. And the harness observes the trace through three channels, none self-reported:

Session transcripts Agentic runs go through claude -p; the session transcript is read back from disk. Task tool calls are SPAWN events, carrying the full WITH payload.
MCP tool log A mock state server (get/set) appends every call to a log file. The set log doubles as a mechanically captured assignment trace.
Canary tokens Each item in a program's data flow carries a unique token. Token presence in worker prompts and final output makes fan-out and aggregation deterministic to score, with no judge.

Here is the raw material every experiment is built from. A generated MDZ program, and the reference trace a deterministic interpreter computes for it – every assignment and every observable output, in order:

program (MDZ)
$v0 = 6
$v1 = $v0 * 5
$v2 = "ok"
IF $v0 != 4 THEN
  IF $v1 != 4 THEN
    Say $v2
  END
END
FOR $v3 IN [1, 4, 0, 3, 7]
  Say $v3
END
reference trace
{ "step": 1, "action": "assign",
  "var": "v0", "value": 6 }
{ "step": 2, "action": "assign",
  "var": "v1", "value": 30 }
{ "step": 3, "action": "assign",
  "var": "v2", "value": "ok" }
{ "step": 4, "action": "emit", "value": "ok" }
{ "step": 5, "action": "emit", "value": "1" }
...

Because the program parses, it has exactly one correct execution. The headline metric wherever traces are compared is unchanged from phase 1: step accuracy, the fraction of the reference trace the model reproduced in order (a model that gets 95 of 100 steps right, in order, scores 0.95). Fuzzy leaves – the prose work a step asks for – are scored by an LLM judge (Claude Sonnet, 40 calls, order counterbalanced) that sees the procedure and the output together; §4 shows the exact question the judge is asked.

Cheap models carry most cells – Claude Haiku and GPT‑5.4 mini, with Sonnet at anchor cells – because a procedure that only runs on a frontier model is doing little work.

The throughline. One mechanism runs through all four results. Notation binds where compliance is locally checkable at the moment of generation – a type annotation contradicting its value, a spawn payload, a decision line. It slips where compliance demands a running discipline the model must hold against its own competence: halting on syntax it has already silently fixed, or journalling state it can track perfectly well in context. That boundary, not program size, is where enforcement has to move from the notation into the harness.
