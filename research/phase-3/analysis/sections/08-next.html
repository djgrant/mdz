Next phase

Phase 3 leaves MDZ's case standing on two legs — state and accountability — and knocks one away from a third. Implications for phase 4:

  • Re-run E2b by reference — done (second run set). With file-backed variables the drift confound is gone and the program tied ralph under an opus orchestrator. Shrinking the payloads did not stop sonnet defecting; only the orchestrator model did.
  • Find a multi-session-sized E2 target. Every current target falls to one sonnet session, so loop-in-harness vs loop-in-orchestrator cannot separate and defection is costless. The mdz-ralph skill is built and compliant; the comparison waits on the target.
  • Enforcement, not request. Sonnet's defection makes the case for moving spawn obligations from notation into the harness (a validator that rejects a run whose trace lacks the declared spawns) — the same notation-vs-harness boundary phase 2 drew for strict mode.
  • Match ralph's budget, then re-ask the E2 question. The cleanest follow-up is the skill run for three sequential map-reduce rounds (matching ralph's session count) and ralph capped to one session — separating shape from spend. If iteration still wins at matched budget, the map-reduce skill should be reshaped around compounding passes with selection as a checkpoint, not a finale: keep ralph's iteration, add the skill's ability to revert a bad pass.
  • Deviation detection becomes the flagship candidate. E1 came back positive-but-second-order rather than null, and its sharpest lesson is that the artefact's value is greatest when the log is poor. The audit-substrate bet held in reserve — can a checker diff a captured trace against the artefact and flag an injected fault — now targets the regime E1 validated: procedures as the thing evidence is interpreted against.
  • Make the E1 log condition harder. Sonnet's ceiling in the log condition means the form effect there is untested, not absent: longer runbooks, noisier logs (interleaved irrelevant events), or partial logs would put headroom back into the cell.
  • Fix the store's assign trace. The store rescues outputs (0.975 at 800) but journals under half the assignments. Chunk-boundary contracts — the orchestrator telling each worker which variables it must persist — would test whether the audit gap is instruction or capacity.
  • Idempotence annotations in the notation. E1's failure-mode split (repeat-and-finish vs skip-and-abandon) suggests a cheap language feature: marking steps as safe or unsafe to re-execute, so a resuming agent can bias towards repeating idempotent steps and never repeating side-effectful ones.
  • Real-world E2 targets. Both sub-experiments used synthetic targets built with known headroom; a real codebase is the deferred test of whether any of the three shapes transfers.
