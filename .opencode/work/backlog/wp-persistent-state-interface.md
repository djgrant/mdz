# Persistent State Interface Design

## Goal/Problem

Define how MDZ skills declare persistent state, without mandating storage backend.

**Reference:** ROADMAP.md Section 9 - Persistent State Interface

## Scope

- Language design
- Spec updates

## Approach

1. Define minimal interface for declaring persistence
2. Design syntax options (frontmatter? inline?)
3. Specify how runtime binds to backends
4. Document schema declaration patterns

## Open Questions

- What's the minimal interface MDZ needs?
- How do skills declare schema for persistent state?
- Should MDZ support querying, or leave to runtime?

## Hypothesis

A clean persistence interface keeps skills portable across environments.
